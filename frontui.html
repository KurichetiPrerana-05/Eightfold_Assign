    This runs entirely in your browser: pdf.js extracts the PDF's text layer, then the same kind of heading-block
    regex this repo's resume_source.py uses (email/phone/github patterns, SKILLS/EDUCATION/EXPERIENCE
    section splitting) pulls out fields — no LLM, no server upload. CSV/ATS JSON are parsed directly. The merge step
    re-implements the conflict-resolution policy from README.md (priority-order names, union'd emails/phones/skills,
    longest-wins headline, max years-of-experience) so you can see the weave for any resume you drop in, not just
    the three bundled samples.
